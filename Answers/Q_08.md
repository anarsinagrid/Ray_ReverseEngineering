# Why Ray Uses Implicit Session Context Capture (`get_session()`)

Ray Train/Tune injects distributed context — trial IDs, topology ranks, dataset shards, reporting handles — directly into each worker process rather than threading it through function signatures. Zero-argument accessors surface this state on demand.

**Core Invariant:** Each worker must observe only its own trial's session context. Shared or leaked session state misattributes metrics and logs to the wrong trial.

## 1. Process-Local Singletons

Ray Train/Tune maintains a module-level `_session` variable ([session.py#L525-L527](https://github.com/ray-project/ray/tree/master/python/ray/train/_internal/session.py))

```python
# Global _TrainSession object initialized by Ray Tune function trainables
# and Ray Train V1 workers.
_session: Optional[_TrainSession] = None
```

Because each worker is a separate process, this singleton is naturally scoped to that worker. `get_session()` exposes the reference ([session.py#L612-L613](https://github.com/ray-project/ray/tree/master/python/ray/train/_internal/session.py))

```python
def get_session() -> Optional[_TrainSession]:
    return _session
```

`_TrainSession` holds a `TrialInfo` dataclass with identity and path fields ([session.py#L55-L67](https://github.com/ray-project/ray/tree/master/python/ray/train/_internal/session.py))

```python
@dataclass

```

plus distributed training rank fields ([session.py#L148-L153](https://github.com/ray-project/ray/tree/master/python/ray/train/_internal/session.py))


These are exposed as typed properties rather than raw environment variables ([session.py#L475-L497](https://github.com/ray-project/ray/tree/master/python/ray/train/_internal/session.py))

## 2. Injected at the Framework Boundary

**Function Trainables:**
`FunctionTrainable.setup()` calls `init_session()` with a `TrialInfo` built from Tune's internal state ([function_trainable.py#L42-L64](https://github.com/ray-project/ray/tree/master/python/ray/tune/trainable/function_trainable.py))

```python
    def setup(self, config):
        init_session(
            training_func=lambda: self._trainable_func(self.config),
            trial_info=TrialInfo(
                name=self.trial_name,
                id=self.trial_id,
                resources=self.trial_resources,
                logdir=self._storage.trial_driver_staging_path,
                driver_ip=None,
                driver_node_id=None,
                experiment_name=self._storage.experiment_dir_name,

        )
```

## 3. Metadata Propagation & Misuse Protection

Checkpoint metadata is merged at report time without user intervention ([session.py#L461-L469](https://github.com/ray-project/ray/tree/master/python/ray/train/_internal/session.py))

```python
        if persisted_checkpoint and self.metadata:
            user_metadata = persisted_checkpoint.get_metadata()
            for k, v in self.metadata.items():
                if k not in user_metadata:
                    user_metadata[k] = v
            persisted_checkpoint.set_metadata(user_metadata)
```

The coordinator receives this in `_train_coordinator_fn` ([base_trainer.py#L92-L102](https://github.com/ray-project/ray/tree/master/python/ray/train/base_trainer.py))

```python
def _train_coordinator_fn(config, trainer_cls, metadata):
    assert metadata is not None
    get_session().metadata = metadata
```

`_in_tune_session()` distinguishes Tune-only sessions from Train sessions ([trainable_fn_utils.py#L59-L60](https://github.com/ray-project/ray/tree/master/python/ray/tune/trainable/trainable_fn_utils.py))

```python
def _in_tune_session() -> bool:
    return get_session() and get_session().world_rank is None
```

Calling session APIs outside an active session is caught by `@_warn_session_misuse()`, which warns and returns a default value rather than crashing ([session.py#L665-L686](https://github.com/ray-project/ray/tree/master/python/ray/train/_internal/session.py))

```python
def _warn_session_misuse(default_value=None):
    def inner(fn):
        @functools.wraps(fn)
        def wrapper(*args, **kwargs):
            session = get_session()
            if not session:
                if log_once(f"{SESSION_MISUSE_LOG_ONCE_KEY}-{fn.__name__}"):
                    warnings.warn(...)
                return default_value
            return fn(*args, **kwargs)
        return wrapper
    return inner
```

## Conclusion

Implicit context capture turns deep orchestration plumbing into flat, zero-argument API calls. By binding session state at the process level, Ray eliminates context parameter threading while maintaining accurate per-trial isolation across distributed training configurations.
