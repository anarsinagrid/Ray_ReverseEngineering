# Why Ray Rejects `async def` Remote Functions

Ray draws a hard boundary between stateless remote tasks and asynchronous execution. As established in [PEP 492](https://peps.python.org/pep-0492/), Python `async`/`await` requires an event loop to schedule coroutine suspension. A standard Ray task has no event loop; if a coroutine were returned, it would be silently serialized into the Object Store as an unresolved object.

**Core Invariant:** A remote task must resolve a concrete return value. Returning an unawaited coroutine is a silent semantic failure, so Ray blocks this at decoration time.

## 1. Remote Tasks Have No Event Loop

`RemoteFunction.__init__` intercepts `async def` functions using `inspect.iscoroutinefunction()` and raises a `ValueError` immediately ([remote_function.py#L98-L103](https://github.com/ray-project/ray/tree/master/python/ray/remote_function.py))

```python
        if inspect.iscoroutinefunction(function):
            raise ValueError(
                "'async def' should not be used for remote tasks. You can wrap the "
                "async function with `asyncio.run(f())`. See more at:"
                "https://docs.ray.io/en/latest/ray-core/actors/async_api.html "
            )
```

This is tested explicitly ([test_asyncio.py#L333-L338](https://github.com/ray-project/ray/tree/master/python/ray/tests/test_asyncio.py))

```python
def test_async_function_errored(ray_start_regular_shared):
    with pytest.raises(ValueError):

        @ray.remote
        async def f():
            pass
```

The normal task execution path in C++ assigns `execution_info.function` directly to `function_executor` with no event loop involvement ([_raylet.pyx#L1746-L1755](https://github.com/ray-project/ray/tree/master/python/ray/_raylet.pyx))

```python
    if <int>task_type == <int>TASK_TYPE_NORMAL_TASK:
        next_title = "ray::IDLE"
        function_executor = execution_info.function
```

If an `async def` bypassed the decorator check, the runtime would serialize the unawaited coroutine into the Object Store and complete successfully while silently dropping the actual payload.

## 2. Async Lifecycle Requires Actors

Event loops need stable persistence, cancellation semantics, and a dedicated lifecycle that is incompatible with disposable task pools.

Async actors get persistent event loops, initialized per concurrency group when Ray detects async methods ([_raylet.pyx#L2132-L2135](https://github.com/ray-project/ray/tree/master/python/ray/_raylet.pyx))

```python
        # Initialize eventloops for asyncio for this actor.
        if core_worker.current_actor_is_asyncio():
```

In the async actor path, `function_executor` routes coroutines through `run_async_func_or_coro_in_event_loop` ([_raylet.pyx#L1762-L1798](https://github.com/ray-project/ray/tree/master/python/ray/_raylet.pyx))

```python
        def function_executor(*arguments, **kwarguments):
            func = execution_info.function

            if core_worker.current_actor_is_asyncio():
                # ...
                return core_worker.run_async_func_or_coro_in_event_loop(
                    async_function, function_descriptor, ..., func_kwargs=kwarguments)
```

The C++ caller yields via `YieldCurrentFiber` while the background event loop executes the coroutine, allowing the actor to multiplex without blocking the runtime.

## 3. Circumvention & Tradeoffs

**The Supported Workaround:**
Documented in [async_api.rst#L313-L334](https://github.com/ray-project/ray/tree/master/doc/source/ray-core/actors/async_api.rst)

```rst
AsyncIO for Remote Tasks
------------------------

We don't support asyncio for remote tasks. Instead, you can wrap the `async`
function with a wrapper to run the task synchronously:

    async def f():
        pass

    @ray.remote
    def wrapper():
        import asyncio
        return asyncio.run(f())
```

**Concurrency Bottleneck:**
Async actors scale concurrent tasks via `max_concurrency` without true parallelism. If all queued tasks become CPU-bound without yielding, the event loop starves.

## Summary

![Q4](Images/Q4.png)

| Aspect                  | Normal Remote Task                        | Async Actor Method                                |
| ----------------------- | ----------------------------------------- | ------------------------------------------------- |
| **Worker Lifecycle**    | Stateless, short-lived                    | Long-lived, stateful                              |
| **Event Loop**          | ❌ None                                   | ✅ Persistent per concurrency group               |
| **async def Support**   | ❌ Blocked at decoration time             | ✅ Fully supported                                |
| **Coroutine Execution** | Would silently return unawaited coroutine | Dispatched via `asyncio.run_coroutine_threadsafe` |

Ray rejects `async def` tasks to protect stateless predictability. Failing at the decorator prevents an entire class of silent runtime errors.
